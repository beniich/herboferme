## 2025-05-15 - [Authentication & Authorization Bypass]
**Vulnerability:** The `/api/upload` endpoint was completely unauthenticated, allowing any user to upload files to the server. Additionally, the admin routes in `src/routes/admin.ts` used a mock `adminOnly` middleware that called `next()` regardless of the user's role, effectively bypassing all RBAC for those routes.
**Learning:** Security middleware was sometimes "simulated" during development to speed up testing, but these mocks were left in the production route definitions. Endpoints like file uploads are often overlooked when they are added as utility routes.
**Prevention:** Never use mock security middleware that defaults to "allow". Use a single source of truth for security middleware (e.g., `src/middleware/security.ts`) and ensure all new routes are explicitly wrapped in `authenticate` and appropriate `authorize` or `requireRole` calls. Added a security-specific test suite `src/upload-security.test.ts` to catch such regressions.
